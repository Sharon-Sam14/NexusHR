# NexusHR REST API Reference Documentation

All endpoints are prefixed with `/api`. Protected API requests require a JWT token passed in the `Authorization` header as: `Bearer <token>`.

## đź”’ Authentication APIs

### 1. Register Account
*   **Endpoint:** `POST /auth/register`
*   **Access:** Public
*   **Request Body:**
    ```json
    {
      "name": "Bob Smith",
      "email": "hr@nexushr.com",
      "password": "password123",
      "role": "HR"
    }
    ```
*   **Success Response (200 OK):**
    ```json
    {
      "token": "eyJhbGciOiJIUzI1NiJ9...",
      "name": "Bob Smith",
      "email": "hr@nexushr.com",
      "role": "HR",
      "userId": 2,
      "employeeId": null
    }
    ```

### 2. Login Account
*   **Endpoint:** `POST /auth/login`
*   **Access:** Public
*   **Request Body:**
    ```json
    {
      "email": "admin@nexushr.com",
      "password": "admin123"
    }
    ```
*   **Success Response (200 OK):**
    ```json
    {
      "token": "eyJhbGciOiJIUzI1NiJ9...",
      "name": "System Administrator",
      "email": "admin@nexushr.com",
      "role": "ADMIN",
      "userId": 1,
      "employeeId": null
    }
    ```

## đź“‚ Employee & Onboarding Document APIs

### 1. List Employees
*   **Endpoint:** `GET /employees`
*   **Access:** Protected (`ADMIN`, `HR`)

### 2. Get Employee by ID
*   **Endpoint:** `GET /employees/{id}`
*   **Access:** Protected (`ADMIN`, `HR`, or Resource Owner)

### 3. Add Employee
*   **Endpoint:** `POST /employees`
*   **Access:** Protected (`ADMIN`, `HR`)

### 4. Upload Onboarding Document
*   **Endpoint:** `POST /employees/{employeeId}/documents`
*   **Access:** Protected (`ADMIN`, `HR`, or Resource Owner)
*   **Request Type:** `multipart/form-data`
*   **Parameters:** `file` (Multipart File)
*   **Success Response (200 OK):**
    ```json
    {
      "id": 5,
      "employeeId": 1,
      "fileName": "employment_contract.pdf",
      "fileType": "application/pdf",
      "fileSize": 124536,
      "uploadDate": "2026-05-28"
    }
    ```

### 5. Download Document
*   **Endpoint:** `GET /employees/{employeeId}/documents/{documentId}/download`
*   **Access:** Protected (`ADMIN`, `HR`, or Resource Owner)
*   **Response:** File byte stream (attachment header)

### 6. Delete Document
*   **Endpoint:** `DELETE /employees/{employeeId}/documents/{documentId}`
*   **Access:** Protected (`ADMIN`, `HR`, or Resource Owner)

## đź’ł Payroll & Payslip Export APIs

### 1. Process Monthly Payroll
*   **Endpoint:** `POST /payroll/generate`
*   **Access:** Protected (`ADMIN`, `HR`)
*   **Request Body:**
    ```json
    {
      "employeeId": 1,
      "month": 5,
      "year": 2026,
      "basicSalary": 95000.0,
      "bonus": 2000.0,
      "deductions": 500.0
    }
    ```
    *(Note: Tax values are automatically generated by the backend engine based on salary tiers if omitted).*

### 2. Get Employee Payroll History
*   **Endpoint:** `GET /payroll/employee/{employeeId}`
*   **Access:** Protected (`ADMIN`, `HR`, or Resource Owner)

### 3. Download CSV Payslip Report
*   **Endpoint:** `GET /payroll/{id}/download`
*   **Access:** Protected (`ADMIN`, `HR`, or Resource Owner)
*   **Response:** CSV attachment file stream
